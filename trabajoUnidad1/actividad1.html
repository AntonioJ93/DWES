Explica las diferencias entre un desarrollador front-end, back-end y full-stack.

Un desarrollador front se encarga de la presentación y lógica de la capa de la vista de la aplicación web, es decir que hay que hacer cuando se pulsa un botón o se hace clic en un enlace, algunas validaciones de formularios ....

Normalmente dependiendo de la acción que realice el usuario se consumirá un determinado servicio proporcionado por el programador back. Este servicio sería el encargado de realizar las acciones necesarias para proporcionarle a la visa la información requerida mediante consultas, actualizaciones a la BBDD

El programador fullstack sería el encargado de realizar ambas tareas front y back.

¿Qué es un framework CSS? ¿Qué es Bootstrap? ¿Cuántos más framework CSS existen y cuáles son?

Una web moderna exige una interfaz de usuario muy cuidada, atractiva y adaptada a múltiples dispositivos, esto hace que las hojas de estilos CSS sean muy extensas y su desarrollo sea lento y complicado. Los framewoks CSS vienen a simplificar esto ya que nos proporcionan clases CSS para nuestro proyecto con unos estilos predeterminados que nos ayudan a maquetar y dar un aspecto visual muy bueno sin tener que escribir uno a uno todos los estilos necesarios, solamente tendríamos que usar esas clases en las etiquetas de html que queramos

Los frameworks más populares son:

  1. Bootstrap
  2. Materialize
  3. Foundation
¿Qué es WCAG? ¿Por qué es tan importante validar en XHTML, CSS y WCAG 2.1 AAA?

La WCAG es un conjunto de pautas que nos ayudan a desarrollar una web de forma que sea accesible a la mayor parte de la población, nos indica que atributos tenemos que poner para los lectores de pantallas para las personas invidentes o con baja visión, contrastes de colores, fuentes de texto …. La normativa europea obliga a todos los organismos públicos y grandes empresas a tener sus webs accesibles y por tanto cumplir las WCAG

XHTML es una versión de XML basado en HTML, en cuanto a es más estricto que HTML aunque usa sus mismas etiquetas. Pretende avanzar hacia una web semántica donde la información y la forma de presentarla estén claramente separadas. Algunos ejemplos de sintaxis de XHTML:

  • Los elementos vacíos deben cerrarse siempre:

    Incorrecto: <br>

    Correcto: </br> o <br/> o <br />

  • Los nombres de elementos y atributos deben ir en minúsculas.

    Incorrecto: <A HREF="http://www.domname.com">Domname</A>

    Correcto: <a href="http://www.domname.com">Domname</a>

WCAG 2.1 es la última versión de las normas WCAG la triple a (AAA) hace referencia a las normas que hay que cumplir, los niveles van desde A, AA, AAA si tu web tiene el nivel AA debe cumplir también las normas nivel A. Un ejemplo sería el siguiente:

Subtítulos (pregrabados)

Nivel A

Se proporcionan subtítulos para todo el contenido de audio pregrabado en medios sincronizados, excepto cuando los medios son una alternativa de medios para el texto y están claramente etiquetados como tales.

Subtítulos (en vivo)

Nivel AA

Se proporcionan subtítulos para todo el contenido de audio en vivo en medios sincronizados.

Descripción de audio ampliada (pregrabada)

Nivel AAA

Cuando las pausas en el audio de primer plano son insuficientes para permitir que las descripciones de audio transmitan el sentido del video, se proporciona una descripción de audio ampliada para todo el contenido de video pregrabado en medios sincronizados.

Explica con tus propias palabras qué es un servidor web, qué es el protocolo HTTP, busca ejemplos donde se ejecuten los métodos HTTP GET y POST y explica qué es Apache y qué es XAMPP.

Es un dispositivo encargado de ejecutar una aplicación web y atender a las peticiones de los clientes, también nos proporciona herramientas como el uso de sesiones, roles …

HTTP es un protocolo para la transmisión de documentos de hipermedia como HTML, XHTML… El cliente hace una petición HTTP al servidor y este le envía la información requerida, HTTP no guarda ningún estado de una petición a otra

HTTP GET es un tipo de petición que se debe usar para solicitar datos al servidor, no para enviar, ya que se envían en la url

HTTP POST es un tipo de petición que se debe usar para enviar datos al servidor, puesto que los envia en la cabecera y no en la url

Apache es un servidor web de código abierto, multiplataforma desarrollado por Apache Software Foundation, es el más usado y tiene una comunidad enorme.

XAMPP es un pack de aplicaciones para el desarrollo web, las aplicaciones son: Apache, MySQL/MariaDB, PHP y Perl. Con este pack se hace muy fácil y cómodo desarrollar aplicaciones web.
